[4\.	第4章 信息校验型反爬虫	1](#_104)

[4\.1\.	信息校验中的“信息”指的是客户端发起网络请求时的请求头和请求正文，而“校验”指的是服务器端通过对信息的正确性、完整性或唯一性进行验证或判断	1](#_156)

[4\.2\.	服务器通过校验请求头或者请求正文中特定的信息，就可以区分正常用户和爬虫程序了	2](#_157)

[4\.3\.	1、User\-Agent反爬虫	2](#_123)

[4\.3\.1\.	1\.User\-Agent反爬虫指的是服务器端通过校验请求头中德UserAgent值来区分正常用户和爬虫程序的手段	2](#_158)

[4\.3\.2\.	2\.User\-Agent就是请求头域之一，服务器能够从User\-Agent对应的值中识别客户端使用的操作系统、CPU类型、浏览器引擎、操作系统语言等	2](#_159)

[4\.3\.3\.	3\.User\-Agent头域并非不可缺少，而且它的值可以被更改	2](#_160)

[4\.3\.4\.	4\.只要请求头中的User\-Agent头域值包含黑名单中德关键词，那么这个请求就无法通过校验	2](#_161)

[4\.4\.	2、Cookie反爬虫	2](#_124)

[4\.4\.1\.	1\.Cookie反爬虫指的是服务器端通过校验请求头中的Cookie值来区分正常用户和爬虫程序的手段	2](#_162)

[4\.4\.2\.	2\.Cookie不仅可以用于Web服务器的用户身份信息存储或状态保持，还能够用于反爬虫	2](#_163)

[4\.4\.3\.	3\.服务器会校验每个请求头中的Cookie值是否符合规则，如果通过校验，则返回正常资源，否则将请求重定向到首页	2](#_164)

[4\.4\.4\.	4\.除了反爬虫外，重定向和Cookie常常被用来指定网站的入口或提高访问门槛	2](#_165)

[4\.4\.5\.	5\.User\-Agent和Cookie都是请求头的默认头域，在值的设定方面有一定的局限性，但是与JavaScript结合后，就会变得很灵活	2](#_166)

[4\.5\.	3、签名验证反爬虫	2](#_125)

[4\.5\.1\.	1\.签名是根据数据源进行计算或加密的过程，签名的结果是一个具有惟一性和一致性的字符串	3](#_167)

[4\.5\.2\.	2\.与Cookie、User\-Agent、Host和Referer等请求头域不同，用于签名验证的信息通常被放在请求正文中发送到服务器端	3](#_168)

[4\.5\.3\.	3\.签名验证有很多实现方式，但原理都是相同的：由客户端生成一些随机值和不可逆的MD5加密字符串，并在发起请求时将这些值发送给服务器端。服务器端使用相同的方式对随机值进行计算以及MD5加密，如果服务器端得到的MD5与前端提交的MD5值相等，就代表正常请求，否则返回403	3](#_169)

[4\.5\.4\.	4\.时间戳可以有效的避免请求正文被复用，组合字符串的消息摘要则用于避免服务器端处理被篡改过的数据	3](#_170)

[4\.5\.5\.	5\.签名验证反爬虫被广泛应用于Web领域，当发现请求正文中包含消息摘要值（特别是32位随机字符串）时，就可以大胆猜测目标网站使用了签名验证反爬虫	3](#_171)

[4\.6\.	4、WebSocket握手验证反爬虫	3](#_126)

[4\.6\.1\.	1\.所有基于HTTP协议的反爬虫都可以用在WebSocket协议上	3](#_172)

[4\.6\.2\.	2\.客户端按照WebSocket规范生成握手信息并向服务器端发送握手请求，然后读取服务器端推送消息，最后验证握手结果	3](#_173)

[4\.7\.	5、WebSocket消息验证反爬虫	3](#_127)

[4\.7\.1\.	1\.WebSocket只需要完成一次握手，就可以保持长期连接，在后续的消息互发阶段是不需要用到HTTP协议的	3](#_174)

[4\.7\.2\.	2\.其实消息互发阶段也是可以对客户端身份进行校验的，这是因为客户端所获取的消息是由服务器端主动推送的	3](#_175)

[4\.8\.	6、WebSocket Ping反爬虫	3](#_128)

[4\.8\.1\.	1\.WebSocket协议规范中约定，服务器端可以向客户端发送Ping帧，当客户端收到Ping帧时应当回复Pong帧。如果客户端不回复或者回复的并不是Pong帧，那么服务器端就可以认为客户端异常，主动关闭该连接	4](#_176)

[4\.9\.	信息校验主要解决了客户端身份鉴别、数据来源判断和请求的合法性判断等问题，避免数据接收者使用篡改过的数据，保证数据的有效性	4](#_177)

[5\.	第5章 动态渲染反爬虫	4](#_105)

[5\.1\.	1、常见的动态渲染反爬虫案例	4](#_129)

[5\.1\.1\.	1\.动态渲染被广泛应用在Web网站中，大部分网站会使用JavaScript来提升用户体验	4](#_178)

[5\.2\.	2、动态渲染的通用解决办法	4](#_130)

[5\.2\.1\.	1\.确实有很多的渲染工具可以满足动态网页爬取需求，其中最常用的是Puppeteer、Splash和Selenium	4](#_179)

[5\.2\.2\.	2\.Selenium是一个用于测试Web应用程序的工具，可以通过Selenium和浏览器驱动调用浏览器执行特定的操作，如发起请求、点击操作、鼠标下滑等	4](#_180)

[5\.2\.3\.	3、使用Python中德异步库来编写爬虫时，Selenium就不是那么适合了。Puppeteer是一个能够做到与Selenium套件相同工作并且支持异步的渲染工具	4](#_181)

[5\.2\.4\.	4\.Splash是一个异步的JavaScript渲染服务，它是带有HTTP API的轻量级Web浏览器，能够并行的处理多个页面请求，在页面上下文中执行自定义的JavaScript以及模拟浏览器中德点击、下滑等操作	4](#_182)

[5\.2\.5\.	5\.Splash返回3种不同类型的结果：页面截图、资源加载信息和HTML文本	4](#_183)

[5\.2\.6\.	6\.Splash命令框使用的并不是Python语言，而是Lua语言	5](#_184)

[5\.2\.7\.	7\.渲染工具知识扩展：Selenium套件通过驱动浏览器执行操作，本质上使用浏览器；Puppeteer实际上通过API控制Chromium或Chrome浏览器；Splash基于开源的浏览器引擎WebKit	5](#_185)

[5\.3\.	使用渲染工具时，不仅要考虑渲染性能，还需要关注页面渲染质量	5](#_186)

[6\.	第6章 文本混淆反爬虫	5](#_106)

[6\.2\.	常见的文本混淆手段有图片伪装、文字映射和自定义字体等	5](#_188)

[6\.3\.	1、图片伪装反爬虫	5](#_131)

[6\.3\.1\.	1\.常见的电话并不是一串数字，而是一张图片	5](#_189)

[6\.3\.2\.	2\.光学字符识别是应对图片伪装反爬虫的最佳选择	5](#_190)

[6\.4\.	2、CSS偏移反爬虫	5](#_132)

[6\.4\.1\.	1\.CSS偏移反爬虫指的是利用CSS样式将乱序的文字排版为人类正常阅读顺序的行为	5](#_191)

[6\.4\.2\.	2\.实际上并不是我们的逻辑和代码有错，而是页面显示错误	5](#_192)

[6\.4\.3\.	3\.CSS样式可以改变页面显示，但这种“改变”仅存在于浏览器（能够解释CSS的渲染工具）中	5](#_193)

[6\.5\.	3、SVG映射反爬虫	5](#_133)

[6\.5\.1\.	1\.利用SVG实现的反爬虫手段用矢量图形代替具体的文字，不会影响用户正常阅读，但爬虫程序无法像读取文字那样获得SVG图形中的内容	5](#_194)

[6\.5\.2\.	2\.映射关系不可能凭空出现，一定使用了某种技术特性	6](#_195)

[6\.5\.3\.	3\.SVG映射反爬虫利用了浏览器与编程语言在渲染方面的差异，以及SVG与CSS定位这样的前端知识	6](#_196)

[6\.6\.	4、字体反爬虫	6](#_134)

[6\.6\.1\.	1\.从网页源代码中看到的并不是符号，而是由&\#X开头的一些字符	6](#_197)

[6\.6\.2\.	2\.字体文件中不仅包含字形数据和点信息，还包括字符到字形映射、字体标题、命名和水平指标等	6](#_198)

[6\.6\.3\.	3\.只有起止坐标和点坐标数据完全相同的字形，描述的才是相同字符	6](#_199)

[6\.6\.4\.	4\.如果开发者频繁改动字体文件或者准备多套字体文件并随机切换，那真是一件头疼的事	6](#_200)

[6\.6\.5\.	5\.绕过系列工作：准备基准字形描述信息；访问目标网页；从目标网页中读取字体编码字符；下载WOFF文件并用Python代码打开；根据字体编码字符找到WOFF文件中的字形轮廓信息；将该自行轮廓信息与基准字形轮廓信息进行对比；得出对比结果；	6](#_201)

[6\.7\.	5、文本混淆反爬虫通用解决方法	6](#_135)

[6\.7\.1\.	1\.实际上我们可以根据需求将页面中所需的部分数据截图保存，然后再用光学字符识别的手段从截图中提取文字	6](#_202)

[6\.7\.2\.	2\.PyTesseract库并非屡试不爽，他能够精确的识别没有干扰信息、轮廓清晰的数字，但对于模糊、有干扰因素的图片以及汉字的识别率就很低了	6](#_203)

[6\.7\.3\.	3\.还可以借助第三方的力量，使用识别率高的API	6](#_204)

[6\.8\.	主动型反爬虫，即使借助渲染工具也无法获得目标数据，那正确的做法：探寻反爬虫原理，用代码实现对应的算法或逻辑	7](#_206)

[7\.	第7章 特征识别反爬虫	7](#_107)

[7\.1\.	特征识别反爬虫是指通过客户端的特征、属性或用户行为特点来区分正常用户和爬虫程序的手段	7](#_207)

[7\.2\.	1、WebDriver识别	7](#_136)

[7\.2\.1\.	1\.使用Navigator对象的webDriver属性来判断客户端是否通过webDriver驱动浏览器	7](#_208)

[7\.2\.2\.	2\.navigator\.webdriver只适用于使用WebDriver的渲染工具，对于Splash这种使用WebKit内核开发的渲染工具来说是无效的	7](#_209)

[7\.2\.3\.	3\.WebDriver检测的结果有3种，分别是true、false和undefined	7](#_210)

[7\.2\.4\.	4\.由于navigator\.webdriver值允许被改变，所以这种检测方法并不可靠	7](#_211)

[7\.3\.	2、浏览器特征	7](#_137)

[7\.3\.1\.	1\.除了Navigator对象的userAgent、cookieEnable、platform、plugins等属性外，Screen对象的一些属性也可以作为判断依据	7](#_212)

[7\.3\.2\.	2\.事实上，只要有可能出现不同结果的属性，就可以作为客户端特性	7](#_213)

[7\.3\.3\.	3\.属性值作为特征并不代表服务器端通过单个属性值就能确认客户端身份，他们只是服务器端判断客户端身份的依据之一	7](#_214)

[7\.4\.	3、爬虫特征	7](#_138)

[7\.4\.1\.	1\.正常用户浏览网页的频率不会像爬虫程序那么高，开发者可以将访问频率过高的客户端视为爬虫程序	8](#_215)

[7\.4\.2\.	2\.面对根据IP地址实现的访问频率限制反爬虫，我们可以使用多台机器共同爬取	8](#_216)

[7\.4\.3\.	3\.要限制爬虫程序的请求频率，首先要找到并确定客户端的身份标识，然后根据标识记录该客户端的请求次数，并且拒绝单位时间内请求次数过多的客户端请求	8](#_217)

[7\.4\.4\.	4\.使用nginx实现根据IP地址限制爬虫访问频率的功能	8](#_218)

[7\.4\.5\.	5\.浏览器指纹也成为客户端指纹，是指由多种客户端特征信息组成的字符串结果。组成浏览器指纹的特征信息包括硬件信息（如屏幕的分辨率和色值、CPU的核心数与类型等）、浏览器信息（插件列表和User\-Agent属性值等）和不可重复信息（如IP地址、已登录用户的cookie等）	8](#_219)

[7\.4\.6\.	6\.不同浏览器一般使用不同的图像处理引擎、图像导出选项、图像压缩级别，即使是使用相同的绘制代码，得出的结果也会有所不同	8](#_220)

[7\.4\.7\.	7\.单一的Canvas指纹、WebGL指纹和Navigator对象属性都不能作为客户端的身份标识，但将这些指纹与属性值组合在一起，就能够降低指纹重复的概率	8](#_221)

[7\.4\.8\.	8\.限制客户端访问频率的前提是找到能够代表客户端身份的特征	8](#_222)

[7\.5\.	4、隐藏链接反爬虫	8](#_139)

[7\.5\.1\.	1\.隐藏链接反爬虫指的是网页中隐藏用于检测爬虫程序的链接的手段。被隐藏的链接不会显示在页面中，正常用户无法访问，但爬虫程序有可能将该链接放到待爬队列，并向该链接发起请求	8](#_223)

[7\.5\.2\.	2\.因为客户端的IP地址会随着网络的关闭与开启而变化，所以在实际应用中，为了避免误伤到正常用户，IP的封禁并不会持续很久	8](#_224)

# <a id="_____32"></a><a id="_104"></a>第4章 信息校验型反爬虫

## <a id="_____34"></a><a id="_156"></a>信息校验中的“信息”指的是客户端发起网络请求时的请求头和请求正文，而“校验”指的是服务器端通过对信息的正确性、完整性或唯一性进行验证或判断

## <a id="_____36"></a><a id="_157"></a>服务器通过校验请求头或者请求正文中特定的信息，就可以区分正常用户和爬虫程序了

## <a id="_____38"></a><a id="_123"></a>1、User\-Agent反爬虫

### <a id="_____40"></a><a id="_158"></a>1\.User\-Agent反爬虫指的是服务器端通过校验请求头中德UserAgent值来区分正常用户和爬虫程序的手段

### <a id="_____42"></a><a id="_159"></a>2\.User\-Agent就是请求头域之一，服务器能够从User\-Agent对应的值中识别客户端使用的操作系统、CPU类型、浏览器引擎、操作系统语言等

### <a id="_____44"></a><a id="_160"></a>3\.User\-Agent头域并非不可缺少，而且它的值可以被更改

### <a id="_____46"></a><a id="_161"></a>4\.只要请求头中的User\-Agent头域值包含黑名单中德关键词，那么这个请求就无法通过校验

## <a id="_____48"></a><a id="_124"></a>2、Cookie反爬虫

### <a id="_____50"></a><a id="_162"></a>1\.Cookie反爬虫指的是服务器端通过校验请求头中的Cookie值来区分正常用户和爬虫程序的手段

### <a id="_____52"></a><a id="_163"></a>2\.Cookie不仅可以用于Web服务器的用户身份信息存储或状态保持，还能够用于反爬虫

### <a id="_____54"></a><a id="_164"></a>3\.服务器会校验每个请求头中的Cookie值是否符合规则，如果通过校验，则返回正常资源，否则将请求重定向到首页

### <a id="_____56"></a><a id="_165"></a>4\.除了反爬虫外，重定向和Cookie常常被用来指定网站的入口或提高访问门槛

### <a id="_____58"></a><a id="_166"></a>5\.User\-Agent和Cookie都是请求头的默认头域，在值的设定方面有一定的局限性，但是与JavaScript结合后，就会变得很灵活

## <a id="_____60"></a><a id="_125"></a>3、签名验证反爬虫

### <a id="_____62"></a><a id="_167"></a>1\.签名是根据数据源进行计算或加密的过程，签名的结果是一个具有惟一性和一致性的字符串

### <a id="_____64"></a><a id="_168"></a>2\.与Cookie、User\-Agent、Host和Referer等请求头域不同，用于签名验证的信息通常被放在请求正文中发送到服务器端

### <a id="_____66"></a><a id="_169"></a>3\.签名验证有很多实现方式，但原理都是相同的：由客户端生成一些随机值和不可逆的MD5加密字符串，并在发起请求时将这些值发送给服务器端。服务器端使用相同的方式对随机值进行计算以及MD5加密，如果服务器端得到的MD5与前端提交的MD5值相等，就代表正常请求，否则返回403

### <a id="_____68"></a><a id="_170"></a>4\.时间戳可以有效的避免请求正文被复用，组合字符串的消息摘要则用于避免服务器端处理被篡改过的数据

### <a id="_____70"></a><a id="_171"></a>5\.签名验证反爬虫被广泛应用于Web领域，当发现请求正文中包含消息摘要值（特别是32位随机字符串）时，就可以大胆猜测目标网站使用了签名验证反爬虫

## <a id="_____72"></a><a id="_126"></a>4、WebSocket握手验证反爬虫

### <a id="_____74"></a><a id="_172"></a>1\.所有基于HTTP协议的反爬虫都可以用在WebSocket协议上

### <a id="_____76"></a><a id="_173"></a>2\.客户端按照WebSocket规范生成握手信息并向服务器端发送握手请求，然后读取服务器端推送消息，最后验证握手结果

## <a id="_____78"></a><a id="_127"></a>5、WebSocket消息验证反爬虫

### <a id="_____80"></a><a id="_174"></a>1\.WebSocket只需要完成一次握手，就可以保持长期连接，在后续的消息互发阶段是不需要用到HTTP协议的

### <a id="_____82"></a><a id="_175"></a>2\.其实消息互发阶段也是可以对客户端身份进行校验的，这是因为客户端所获取的消息是由服务器端主动推送的

## <a id="_____84"></a><a id="_128"></a>6、WebSocket Ping反爬虫

### <a id="_____86"></a><a id="_176"></a>1\.WebSocket协议规范中约定，服务器端可以向客户端发送Ping帧，当客户端收到Ping帧时应当回复Pong帧。如果客户端不回复或者回复的并不是Pong帧，那么服务器端就可以认为客户端异常，主动关闭该连接

## <a id="_____88"></a><a id="_177"></a>信息校验主要解决了客户端身份鉴别、数据来源判断和请求的合法性判断等问题，避免数据接收者使用篡改过的数据，保证数据的有效性

## <a id="_____168"></a><a id="_207"></a>特征识别反爬虫是指通过客户端的特征、属性或用户行为特点来区分正常用户和爬虫程序的手段

### <a id="_____172"></a><a id="_208"></a>1\.使用Navigator对象的webDriver属性来判断客户端是否通过webDriver驱动浏览器

### <a id="_____174"></a><a id="_209"></a>2\.navigator\.webdriver只适用于使用WebDriver的渲染工具，对于Splash这种使用WebKit内核开发的渲染工具来说是无效的

### <a id="_____176"></a><a id="_210"></a>3\.WebDriver检测的结果有3种，分别是true、false和undefined

## <a id="_____180"></a><a id="_137"></a>2、浏览器特征

### <a id="_____182"></a><a id="_212"></a>1\.除了Navigator对象的userAgent、cookieEnable、platform、plugins等属性外，Screen对象的一些属性也可以作为判断依据

## <a id="_____284"></a><a id="_152"></a>1、编码与加密

### <a id="_____286"></a><a id="_246"></a>字符集是指各国家的文字、标点符号、图形符号和数字等字符的集合

### <a id="_____288"></a><a id="_247"></a>ASCII是基于拉丁字母表的一套计算机编码系统，主要用于显示现代英语和其他西欧语言

### <a id="_____290"></a><a id="_248"></a>ASCII的RFC文档编码为20，其中约定了ASCII的使用范围、标准码、字符表示和代码识别等内容

### <a id="_____292"></a><a id="_249"></a>ASCII码默认使用7位二进制数来表示所有的大写字母、小写字母、数字（0\-9）、标点符号和特殊的控制符

### <a id="_____294"></a><a id="_250"></a>Base64的RFC文档编码为4648，它约定了Base16、Base32和Base64的编码规范和计算方法

### <a id="_____296"></a><a id="_251"></a>有经验的爬虫工程师看到带有“==”符号或者“=”符号的字符串时，自然会认为这是Base64编码字符串后得到的结果

### <a id="_____298"></a><a id="_252"></a>Base64编码和解码时都是将原本8位的二进制数转换成6位的二进制数

### <a id="_____300"></a><a id="_253"></a>MD5消息摘要算法是一种被广泛使用的散列函数，能够将任意长度的消息转换成128位的消息摘要

### <a id="_____302"></a><a id="_254"></a>MD5的RFC文档编码为1321，约定了一些术语和符号，并描述了MD5算法的计算步骤和方法

### <a id="_____304"></a><a id="_255"></a>MD5的典型应用场景就是一致性验证，如文件一致性和信息一致性

### <a id="_____306"></a><a id="_256"></a>MD5在此场景下的作用是验证下载后得文件是否与服务器端文件一致，及时发现文件缺失现象

### <a id="_____308"></a><a id="_257"></a>MD5算法具有“不可逆”和“压缩”这两种特性，而MD5算法输出的值则具有“不可读”的特性，这也使得其成为密码保存的不二之选

### <a id="_____310"></a><a id="_258"></a>MD5是消息摘要算法中德一种，同样计算消息摘要的算法还有SHA1和SHA256等

### <a id="_____312"></a><a id="_259"></a>加密和解密时使用同一个密钥的加密方式叫做对称加密，使用不同密钥的是非对称加密

### <a id="_____314"></a><a id="_260"></a>对称加密的速度更快，速度的优势使得它更适合大量数据加密的场景。常见的对称加密算法有DES、3DES、BLOWFISH、RC5和AES等

### <a id="_____316"></a><a id="_261"></a>大部分编程语言有成熟的AES库，如JavaScript语言的CryptoJS库，python语言的pycrypto库

### <a id="_____318"></a><a id="_262"></a>与对称加密不同的是，非对称加密方式需要两个密钥：公钥和私钥

### <a id="_____320"></a><a id="_263"></a>在非对称加密算法中，应用最广泛、强度最高的是RSA算法

### <a id="_____322"></a><a id="_264"></a>在学习RSA算法的原理之前，需要了解一些数学概念，如质数、互质关系、欧拉函数、欧拉定理和模反元素

### <a id="_____324"></a><a id="_265"></a>无论是Base64编码、消息摘要算法MD5还是加密算法，都使用一定的计算规则将一串字符转换成不可读的另一串字符

### <a id="_____326"></a><a id="_266"></a>不可逆的消息摘要算法常用于签名，而可逆的对称加密算法AES和非对称加密算法RSA通常用于保护数据
